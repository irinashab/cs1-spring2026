Midterm Project Idea Bank

Your midterm project should demonstrate your understanding of:

  • variables
  • functions
  • arrays or loops
  • animation
  • interaction (keyboard or mouse)

The goal is not to build a huge game. A small, well-organized sketch that uses these ideas clearly is a strong project.

Below are some ideas you may use as inspiration. You may modify them or combine several ideas into one project.

1. Coin Collector

A player moves around the screen collecting coins.

  • Arrow keys move the player
  • Coins appear at random positions
  • Collision increases the score
  • A timer respawns coins

Concepts: movement, collision, timers, scoring

2. Dodge the Enemies

The player must avoid moving obstacles.

  • Enemies move across the screen
  • If the player touches one, the game resets
  • Score increases with survival time

Concepts: timers, movement, collision

3. Animated Character Walk

Create a character that walks across the screen using sprite animation.

  • Use an array of images
  • Animate when the character moves
  • Idle frame when the character stops

Concepts: arrays, animation, keyboard input

4. Reaction Time Game

A circle appears randomly on the screen. The player must click it quickly.

  • A timer measures reaction time
  • Circles appear in random locations
  • Display best score

Concepts: timers, mouse input, randomness

5. Falling Objects Game

Objects fall from the top of the screen and the player must catch them.

  • Basket controlled by arrow keys
  • Objects fall at increasing speed
  • Score increases when caught

Concepts: arrays, loops, collision

6. Memory Flash Game

Shapes flash briefly on the screen and the player must remember the sequence.

  • Timer controls flashing
  • Player repeats sequence with keys
  • Sequence length increases

Concepts: arrays, timers, input

7. Moving Target Game

Targets move around the screen and the player must click them.

  • Targets move randomly
  • Clicking a target gives points
  • Targets respawn after a delay

Concepts: movement, timers, mouse input

8. Simple Platform Jump

Create a character that jumps between platforms.

  • Gravity pulls the character downward
  • Collision with platforms stops falling
  • Arrow keys move left and right

Concepts: physics, collision, movement

9. Maze Explorer

The player moves through a maze trying to reach a goal.

  • Walls block movement
  • Goal area ends the game
  • Timer records completion time

Concepts: collision detection, movement

10. Particle Fireworks

Create animated fireworks that appear when the mouse is clicked.

  • Particles move outward
  • Colors change over time
  • Particles disappear after a short delay

Concepts: loops, timers, animation

11. Animated Story Scene

Create a short animated scene instead of a game.

  • Characters move across the screen
  • Animations triggered by key presses
  • Sound or text narration

Concepts: animation, timing, storytelling

12. Math or Pattern Visualizer

Create an interactive visual pattern.

  • Shapes change based on keyboard input
  • Colors shift over time
  • Arrays store pattern elements

Concepts: loops, arrays, animation

13. Mini Drawing Program


Let the user draw on the screen with the mouse. Use keys to change brush size or color, and store drawing points in arrays.

Note: These are only examples. Creative ideas are welcome.

Please do not submit Pong or Pong-style games. Pong is a classic beginner project, but we have seen it many times before. Try something a bit more original.

Tips for a Strong Project

  • Keep the idea simple.
  • Break your code into functions.
  • Use meaningful variable names.
  • Test your code frequently.
  • Add comments explaining your logic.

A clear and well-organized project is more important than a very complex one.

Things to Avoid

  • Copying projects from the internet
  • Trying to build something too large
  • Writing all code inside draw()
  • Hard-coding numbers instead of using variables

Start Early

Begin with a small working sketch and gradually add features. A project that evolves step by step is much easier to complete.
