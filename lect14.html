Lesson 14: Intro to 3D in Processing

Today’s goals

  • Switch from the default renderer to P3D.
  • Understand 3D coordinates: x, y, and z.
  • Use translate(), rotateX(), rotateY(), and rotateZ().
  • Draw 3D primitives such as box() and sphere().
  • Build custom 3D shapes with beginShape() and vertex().
  • Experiment with lighting and level of detail.

1. Turning on 3D

In Processing, 3D requires the P3D renderer. Without it, you are still in the default 2D world. 

void setup() {
  size(400, 400, P3D);
}

The Processing tutorial points out that P3D is required for actual 3D drawing, and it also gives access to features such as lighting and textures.

2. The coordinate system in 3D

In 2D, we worked with x and y. In 3D, we add z.

  • x: left/right
  • y: up/down on the screen (in Processing, positive y goes down)
  • z: depth

A very important Processing detail: the tutorial uses a left-handed coordinate system. Positive z comes toward the viewer; negative z goes farther away.

This is the first place where students often get confused. “Bigger z” does not mean “higher.” It means “closer to us.”

3. You do not place 3D shapes by adding a z-argument everywhere

In 2D, students are used to code like rect(x, y, w, h). In 3D, we usually move the coordinate system first and then draw the shape around the new origin. 

translate(width/2, height/2, 0);
rectMode(CENTER);
rect(0, 0, 100, 100);

The key idea is the same as in 2D transformations, except now translate() can take three arguments. 

translate(x, y, z);

4. First example: a simple box

Processing gives us some built-in 3D primitives. The simplest one is box(). 

// 3D primitive - box

void setup() {
  size(400, 400, P3D);
}

void draw() {
  background(0);
  translate(width/2, height/2);
  // rotateY(frameCount * PI / 60);
  // rotateX(frameCount * PI / 60);
  // rotateZ(frameCount * PI / 60);
  box(150);
}

Notice how the box is centered by translate(). The box itself does not take x, y, and z location arguments. We move the world, then draw the shape.

Try this

  • Uncomment one rotation at a time.
  • Then try two rotations together.
  • Ask yourself: which axis makes the box spin like a ceiling fan? Which makes it tumble forward?

5. Rotations in 3D

In regular 2D Processing, rotate() is really rotation around the z-axis. In 3D we can be more explicit:

  • rotateX(...)
  • rotateY(...)
  • rotateZ(...)
translate(width/2, height/2, 0);
rotateX(PI/8);
rotateY(PI/8);
rotateZ(PI/8);
box(100);

Order matters. If you change the order of the rotations, you may get a different result.

6. Building a cube “by hand”

Built-in shapes are convenient, but they hide the geometry. This example draws a cube edge by edge with 3D lines. 

// 3D cube done from scratch

float theta = 0;

void setup() {
  size(200, 200, P3D);
}

void draw() {
  background(174);
  smooth();
  pushMatrix();
  translate(50, 50);
  // rotateX(theta);
  // rotateY(theta);
  // rotateZ(theta);

  stroke(255, 0, 0);
  strokeWeight(3);

  line(0, 0, 0, 0, 0, -100);
  line(0, 0, -100, 100, 0, -100);
  line(100, 0, -100, 100, 0, 0);
  line(100, 0, 0, 0, 0, 0);

  line(0, 0, 0, 0, 100, 0);
  line(0, 100, 0, 100, 100, 0);
  line(100, 100, 0, 100, 100, -100);
  line(100, 100, 0, 100, 0, 0);

  line(100, 100, -100, 100, 0, -100);
  line(0, 0, -100, 0, 100, -100);
  line(0, 100, 0, 0, 100, -100);
  line(0, 100, -100, 100, 100, -100);

  theta += 0.01;
  popMatrix();
}

This is a nice bridge from 2D to 3D because it shows that a 3D point is just three numbers, and a 3D line needs six coordinates: line(x1, y1, z1, x2, y2, z2).

Teaching note

This sketch is also a good place to review push() and pop(). They keep transformations from affecting everything else in the sketch.

7. Sphere and detail level

Processing also gives us sphere(). One useful companion function is sphereDetail(), which changes how smooth the sphere looks. 

int detail = 10;
int depth = 100;

void setup() {
  size(400, 400, P3D);
}

void draw() {
  background(50);
  translate(width/2, height/2, depth);
  rotateY(PI * frameCount / 125);
  lights();
  fill(100, 50, 175);
  stroke(150, 75, 255);
  sphereDetail(detail);
  sphere(20);
}

void keyPressed() {
  if (key == CODED) {
    if (keyCode == UP) {
      if (depth < 310) {
        depth += 20;
        detail += 2;
      }
    } else if (keyCode == DOWN) {
      if (depth > 70) {
        depth -= 20;
        detail -= 2;
      }
    }
  }
}

In this sketch, the arrow keys change both the distance from the viewer and the number of triangles used to approximate the sphere.

Questions

  • What happens when the detail is very low?
  • How does changing depth affect the illusion of perspective?
  • Why do we call lights() here?

8. Lighting

Without lighting, many 3D shapes look flat. One of the easiest ways to improve the illusion of depth is to add light. 

// There are different kinds of lighting:
// ambientLight(), directionalLight(), spotLight(), pointLight()

size(500, 500, P3D);
background(0);
noStroke();
specular(50);
shininess(5);
emissive(25);
lightSpecular(0, 0, 255);
pointLight(125, 50, 50, 300, 75, 200);
lightFalloff(.001, .01, 0);
ambientLight(0, 40, 0);
translate(width/2, height/2, 0);
sphere(150);

This sketch is a demo rather than a full animation, but it introduces useful vocabulary:

  • ambientLight() adds general background light.
  • pointLight() places a light source at a position in 3D space.
  • specular() and shininess() affect highlights.

Students do not need to memorize all light functions immediately. The main conceptual point is that light helps our eyes read form and depth.

9. Custom 3D shapes with vertices

The tutorial emphasizes that 3D is not limited to box() and sphere(). We can also create our own shapes by combining vertices.

Your example below is especially nice because it connects geometry, arrays, classes, and keyboard interaction. It can behave like a cube, pyramid, cone, or cylinder depending on settings. 

int pts = 4;
float angle = 0;
float radius = 115;
float cylinderLength = 100;
Point3D vertices[][];
boolean isPyramid = true;

void setup() {
  size(400, 400, P3D);
  noStroke();
}

void draw() {
  background(170, 95, 95);
  lights();
  fill(255, 200, 200);
  translate(width/2, height/2);
  rotateX(frameCount * PI / 150);
  rotateY(frameCount * PI / 130);
  rotateZ(frameCount * PI / 180);

  vertices = new Point3D[2][pts + 1];

  for (int i = 0; i < 2; i++) {
    angle = 0;
    for (int j = 0; j <= pts; j++) {
      vertices[i][j] = new Point3D();

      if (isPyramid) {
        if (i == 1) {
          vertices[i][j].x = 0;
          vertices[i][j].y = 0;
        } else {
          vertices[i][j].x = cos(radians(angle)) * radius;
          vertices[i][j].y = sin(radians(angle)) * radius;
        }
      } else {
        vertices[i][j].x = cos(radians(angle)) * radius;
        vertices[i][j].y = sin(radians(angle)) * radius;
      }

      vertices[i][j].z = cylinderLength;
      angle += 360.0 / pts;
    }
    cylinderLength *= -1;
  }

  beginShape(QUAD_STRIP);
  for (int j = 0; j <= pts; j++) {
    vertex(vertices[0][j].x, vertices[0][j].y, vertices[0][j].z);
    vertex(vertices[1][j].x, vertices[1][j].y, vertices[1][j].z);
  }
  endShape();

  for (int i = 0; i < 2; i++) {
    beginShape();
    for (int j = 0; j < pts; j++) {
      vertex(vertices[i][j].x, vertices[i][j].y, vertices[i][j].z);
    }
    endShape(CLOSE);
  }
}

This sketch also relies on a small helper class: 

class Point3D {
  float x, y, z;

  Point3D() {
  }

  Point3D(float x, float y, float z) {
    this.x = x;
    this.y = y;
    this.z = z;
  }
}

Why this example is useful

  • It reinforces that a 3D point is a data object.
  • It shows how trigonometry can generate circular arrangements of points.
  • It uses arrays to organize related geometry.
  • It shows that changing just a few coordinates can radically change the final shape.

10. Important ideas from the tutorial

  • Use P3D when you need 3D, lighting, or certain OpenGL-based effects.
  • In Processing 3D, position is usually controlled with transformations, not by adding extra arguments to every shape function.
  • Positive z comes toward the viewer.
  • Rotations around different axes create very different visual effects.
  • 3D primitives are easy to start with, but custom vertex-based shapes give you far more control.
  • Lighting often makes the difference between “flat object” and “convincing form.”

11. Common mistakes

  • Forgetting to write size(..., P3D).
  • Assuming that rect(), box(), or sphere() take position arguments the same way 2D functions do.
  • Mixing up what z means.
  • Applying transformations without push()/pop() and then wondering why everything moves.
  • Thinking rotation order does not matter.

12. Suggested live-coding sequence

  1. Start with a plain box() in the center.
  2. Add one rotation.
  3. Add a second rotation.
  4. Replace the box with a sphere() and call sphereDetail().
  5. Add lights().
  6. Show one custom line-based or vertex-based shape.

13. Practice problems

  1. Make a box spin around two axes at different speeds.
  2. Create two spheres: one close to the viewer and one far away.
  3. Modify the hand-built cube so it rotates continuously.
  4. Change the custom shape sketch so the number of points can go below 4. What breaks, and why?
  5. Add a key that switches between box() and sphere().
  6. Experiment with light color and position. Which changes are most noticeable?

14. Reflection prompt

What feels genuinely new in 3D compared to 2D: the extra coordinate, the transformations, the lighting, or the mental challenge of imagining the object in space?
