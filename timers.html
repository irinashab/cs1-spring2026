Timers in Processing

Many interactive sketches need to measure time. For example:

  • Spawn an enemy every few seconds
  • Display a message for a short time
  • Delay an animation
  • Make objects appear or disappear

Processing provides two simple ways to measure time:

  • frameCount – counts frames
  • millis() – counts milliseconds since the sketch started

Timer using frameCount

The variable frameCount increases by 1 every time the screen updates.

If your sketch runs at about 60 frames per second:

  • 60 frames ≈ 1 second
  • 120 frames ≈ 2 seconds
  • 300 frames ≈ 5 seconds

Example

Display a circle after about 2 seconds.


void setup(){
  size(400,400);
}

void draw(){

  background(240);

  if(frameCount > 120){
    ellipse(200,200,80,80);
  }

}

After roughly 120 frames the circle appears.

Starting a frame timer

Often we want to measure time starting from a specific moment. 


int startFrame;

void setup(){
  size(400,400);
  startFrame = frameCount;
}

void draw(){

  background(240);

  if(frameCount - startFrame > 180){
  fill(0);
    text("3 seconds passed",150,200);
  }

}

The expression 


frameCount - startFrame

tells us how many frames have passed since the timer started.

A helper function using frameCount

Instead of repeating the same logic, we can write a function. 


boolean frameTimer(int startFrame, int framesNeeded){
  return frameCount - startFrame >= framesNeeded;
}

Example


int startFrame;

void setup(){
  size(400,400);
  startFrame = frameCount;
}

void draw(){

  background(240);

  if(frameTimer(startFrame,180)){
    fill(0);
    text("3 seconds passed",150,200);
  }

}

Timer using millis()

The function millis() returns the number of milliseconds since the sketch started.

1000 milliseconds = 1 second

Example


void draw(){

  background(240);

  if(millis() > 2000){
    ellipse(200,200,80,80);
  }

}

The circle appears after about 2000 milliseconds (2 seconds).

Starting a millis timer


int startTime;

void setup(){
  size(400,400);
  startTime = millis();
}

void draw(){

  background(240);

  if(millis() - startTime > 3000){
    fill(0);
    text("3 seconds passed",150,200);
  }

}

The expression 


millis() - startTime

tells us how much time has passed since the timer started.

A reusable millis timer function


boolean timePassed(int startTime, int duration){
  return millis() - startTime >= duration;
}

Example


int startTime;

void setup(){
  size(400,400);
  startTime = millis();
}

void draw(){

  background(240);

  if(timePassed(startTime,3000)){
    fill(0);
    text("3 seconds passed",150,200);
  }

}

Repeating timer (very useful pattern)

Sometimes we want an event to happen repeatedly every few seconds. 


int lastSpawn;

void setup(){
  size(400,400);
  lastSpawn = millis();
}

void draw(){

  if(millis() - lastSpawn > 2000){
    println("spawn!");
    lastSpawn = millis();
  }

}

This prints a message every 2 seconds.

Which timer should I use?

Timer Best use
frameCount Animations tied to frames
millis() Real-world time delays

In most cases millis() is more reliable because it does not depend on the frame rate.

Avoid delay()

Processing has a function called delay(): 


delay(2000);

However, this stops the entire program, freezing animation and input.

Timers based on frameCount or millis() allow the sketch to keep running smoothly.

Mini practice

  1. If your sketch runs at 60 FPS, how many frames correspond to about 5 seconds?
  2. What value of millis() corresponds to 3 seconds?
  3. Modify the repeating timer so that the event happens every 5 seconds.

Summary

Timers allow sketches to react to the passage of time without stopping the program.

  • frameCount counts frames.
  • millis() measures real time in milliseconds.
  • Subtracting start time from current time gives elapsed time.

These simple patterns are widely used in interactive graphics and games.
