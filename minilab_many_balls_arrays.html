Mini-Lab: Many Bouncing Balls (Arrays)

Purpose

In this mini-lab, you will extend the classic bouncing ball sketch by using arrays. Instead of animating just one ball, you will animate many. This is your first step toward managing multiple moving objects efficiently.

Reference: single bouncing ball sketch (given) 
float xspeed, yspeed;
float xpos, ypos, wdth, ht;

void setup(){
  size(1000, 1000);
  background(0);

  xpos = random(width);
  ypos = random(height);
  xspeed = random(3);
  yspeed = random(3);

  wdth = random(20);
  ht = wdth;

  noStroke();
}

void draw(){
  background(0);
  ellipse(xpos, ypos, wdth, ht);

  xpos += xspeed;
  ypos += yspeed;

  if (xpos >= width || xpos <= 0){
    xspeed *= -1;
  }
  if (ypos >= height || ypos <= 0){
    yspeed *= -1;
  }
}

Instructions

  1. Set the number of balls. Start with: 
    int n;// number of balls
    This value determines the number of balls and the size of your arrays.
  2. Replace single variables with arrays. Every ball property (position, speed, size, etc.) becomes an array of length n.
  3. Initialize all balls in setup(). Use a for-loop so that each index i represents one ball.
  4. Animate all balls in draw(). Use a for-loop to draw, update, and bounce each ball.
  5. Experiment with scalability. Try n = 100, 1000, 10000, and beyond. Write 2–3 sentences: How many balls can your program handle smoothly on your computer?

Personalization (mandatory)

At the top of your sketch, include:

int d1 = int(random(0,10));   // choose a random digit 0–9
int d2 = int(random(0,10));;   // choose a random digit 0–9
int n  = 10 + d1;

Your initialization must use d2 in a visible way (example: ball sizes depend on d2).

Index checks (do not remove)

In setup(), print endpoints so indexing mistakes surface immediately:

println("x0=" + xpos[0] + " xLast=" + xpos[n-1]);
println("w0=" + wdth[0] + " wLast=" + wdth[n-1]);

Off-screen recovery trap (do not remove)

After initialization, force ball 0 to start slightly off-screen:

xpos[0] = -5;

Ball 0 must eventually appear and bounce normally.

Bounce must respect ball size

Your bounce logic must use each ball’s size (radius). Balls should not disappear halfway off-screen.

Loop-bound comment (required)

Above your main update loop, include this comment exactly:

// My loop uses i < n because  - give your reasoning here.
