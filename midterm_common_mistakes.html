Most Common Mistakes

CS1 Midterm: Most Common Mistakes

This is a quick self-check list. If you have made any of these mistakes before, practice until you can catch them immediately.

Drawing / Coordinates

  • Mixing up rect() anchor (top-left) with ellipse() anchor (center).
  • Forgetting where (0,0) is (top-left corner).
  • Not stating what’s drawn vs what’s overwritten when the background redraws each frame.

Conditionals

  • Using = instead of ==.
  • Describing both branches instead of the one that actually runs.
  • Not simplifying boolean expressions (especially double negation).

Variables / Updates

  • Forgetting that a-- means “subtract 1”.
  • Forgetting that a -= 4 means a = a - 4.
  • Thinking assignment “adds” instead of replacing the old value.

Scope / Compilation

  • Trying to use a variable declared in setup() inside draw().
  • Answering “what prints” when the code actually does not compile.
  • Not using precise language: compile-time error vs run-time error.

Loops

  • Accidental semicolon: for(...); (loop does nothing; block runs once).
  • Off-by-one errors: using <= when you need <.
  • Wrong start/end values (especially for even/odd ranges).

Functions

  • Tracing without understanding what the function computes “in general.”
  • Confusing parameter (variable in the header) with argument (value passed in).
  • Forgetting that local variables inside a function disappear after the function returns.

Arrays

  • Using index arr.length (out of bounds) instead of arr.length - 1.
  • Confusing values with indices in nested indexing (e.g., arr[arr[4]]).
  • Forgetting that array indices start at 0.

Random

  • Forgetting that random(n) never reaches n.
  • Forgetting +1 when simulating dice rolls.

Two instant checks before you finalize an answer

  1. Does it compile? (Scope, missing semicolons, undefined names.)
  2. Are indices safe? (Last valid index is length - 1.)
