Putting It Together: A Tiny Game Sketch

By now, we have seen several important tools:

  • movement
  • animation
  • timers
  • collision detection

These ideas are the foundation of many simple games and interactive sketches. In this page, we combine them into one small example.

What this sketch does

In this example:

  • the player moves left and right with the arrow keys
  • the player is animated using an array of images
  • the player collects a coin by touching it
  • after the coin is collected, it reappears 2 seconds later

This is not a full game, but it shows how the pieces work together.

Complete example

int numFrames = 4;
PImage[] walk = new PImage[numFrames];

int playerX = 100;
int playerY = 220;
int playerW = 80;
int playerH = 80;

int frame = 0;

int coinX = 400;
int coinY = 240;
int coinSize = 30;

boolean coinVisible = true;
int coinTimerStart = 0;

int score = 0;

void setup(){
  size(700, 400);

  for(int i = 0; i < numFrames; i++){
    walk[i] = loadImage("walk" + i + ".png");
  }

  textSize(24);
}

void draw(){
  background(220, 235, 250);

  drawGround();
  drawCoin();
  drawPlayer();
  checkCoinCollision();
  respawnCoin();
  drawScore();
}

void drawGround(){
  fill(100, 180, 100);
  rect(0, 300, width, 100);
}

void drawPlayer(){
  image(walk[frame], playerX, playerY, playerW, playerH);
}

void drawCoin(){
  if(coinVisible){
    fill(255, 215, 0);
    ellipse(coinX, coinY, coinSize, coinSize);
  }
}

void drawScore(){
  fill(0);
  text("Score: " + score, 20, 30);
}

void keyPressed(){

  if(keyCode == RIGHT){
    playerX += 10;
    frame = (frame + 1) % numFrames;
  }

  if(keyCode == LEFT){
    playerX -= 10;
    frame = (frame + 1) % numFrames;
  }

}

void checkCoinCollision(){
  if(coinVisible && rectCircle(playerX, playerY, playerW, playerH,
                                 coinX, coinY, coinSize / 2)){
    coinVisible = false;
    coinTimerStart = millis();
    score++;
  }
}

void respawnCoin(){
  if(!coinVisible && millis() - coinTimerStart > 2000){
    coinX = int(random(50, width - 50));
    coinVisible = true;
  }
}

boolean rectCircle(float rx, float ry, float rw, float rh,
                   float cx, float cy, float r){

  float closestX = constrain(cx, rx, rx + rw);
  float closestY = constrain(cy, ry, ry + rh);

  return dist(cx, cy, closestX, closestY) <= r;
}

How the sketch is organized

Notice that the sketch is divided into small functions:

  • drawGround()
  • drawCoin()
  • drawPlayer()
  • checkCoinCollision()
  • respawnCoin()
  • drawScore()

This makes the code easier to read, debug, and improve.

Movement

The player moves when the left or right arrow key is pressed: 

if(keyCode == RIGHT){
  playerX += 10;
}

if(keyCode == LEFT){
  playerX -= 10;
}

Changing playerX moves the character horizontally.

Animation

The walking images are stored in an array: 

PImage[] walk = new PImage[numFrames];

Each key press advances the animation: 

frame = (frame + 1) % numFrames;

The modulo operator keeps the frame index between 0 and numFrames - 1, so the animation loops.

Collision

To decide whether the player touches the coin, we use a rectangle-circle collision test: 

if(coinVisible && rectCircle(playerX, playerY, playerW, playerH,
                             coinX, coinY, coinSize / 2)){
  coinVisible = false;
  coinTimerStart = millis();
  score++;
}

When the player touches the coin:

  • the coin disappears
  • the timer starts
  • the score increases

Timer

The coin comes back 2 seconds later: 

if(!coinVisible && millis() - coinTimerStart > 2000){
  coinX = int(random(50, width - 50));
  coinVisible = true;
}

This uses millis() to measure elapsed time.

Why this matters for your project

Many midterm projects will use this exact structure:

  • a character or object moves
  • something is drawn repeatedly in draw()
  • collisions trigger events
  • timers control delays or respawning

Even if your project is very different in theme, the same programming ideas will still apply.

Ideas for modifications

Here are some simple ways to extend the sketch:

  • add vertical movement
  • add more than one coin
  • add an enemy to avoid
  • give the player 30 seconds to collect coins
  • end the game when the score reaches a target
  • replace the coin with a different collectible

Common mistakes

  • forgetting to put image files in the data folder
  • using the wrong image file names
  • forgetting to update the animation frame
  • checking collision even when the coin is invisible
  • trying to use delay() instead of a timer
  • writing one giant draw() instead of breaking code into functions

Mini challenge

  1. Make the coin respawn in both a random x and a random y position.
  2. Add a second collectible worth 5 points.
  3. Display “You win!” when the score reaches 10.
  4. Add a simple enemy that decreases the score when touched.

Final thought

A good project does not need to be huge. A small sketch that combines movement, animation, timers, and collisions in a clear way is already a strong midterm project.

Focus on making your code organized, readable, and functional.
