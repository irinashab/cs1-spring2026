Lab: Inheritance Options

Goal

In this lab, you will practice inheritance in Processing. You have two paths to choose from. Both paths focus on the same big idea: a child class can inherit data and behavior from a parent class, and then customize what it does.

Choose One Path

Option A: Build on Your Previous Lab

If you already created your own simple class in a previous lab, you may continue working with it. Your task is to turn that old class into a parent class, then create at least one child class that extends it.

Option B: Start from the Shape / Ellipse example

If you prefer, you may use the Shape / Ellipse example from class. Your task is to create another subclass of Shape, such as Rectangle, Triangle, or Square.

What You Must Show

  • A parent class with shared instance variables and at least one shared method.
  • A child class that uses extends.
  • A child constructor that uses super(...).
  • At least one method that is overridden in the child class.
  • A sketch that creates objects and shows that the child class behaves differently from the parent.

Option A: Continue Your Previous Lab

Suppose your earlier lab had a class such as Creature, Bug, Ball, Flower, or something else you designed yourself. You may now reorganize it as follows:

  1. Keep the common data and behavior in the parent class.
  2. Create a child class that extends the parent.
  3. Change the child class so that it overrides at least one method such as display(), move(), or both.

Example idea: if your old class was Bug, you could now make:

class Bug {
  float x, y;
  float speedX, speedY;

  Bug(float x, float y) {
    this.x = x;
    this.y = y;
    speedX = random(-2, 2);
    speedY = random(-2, 2);
  }

  void move() {
    x += speedX;
    y += speedY;
  }

  void display() {
    ellipse(x, y, 20, 20);
  }
}

class FancyBug extends Bug {
  FancyBug(float x, float y) {
    super(x, y);
  }

  void display() {
    fill(255, 0, 0);
    ellipse(x, y, 30, 15);
    line(x - 10, y, x + 10, y);
  }
}

You do not have to use this exact example. Use your own earlier class if you want.

Option B: Start from Shape / Ellipse

Here is a parent class Shape and one child class Ellipse. You may use this as your starting point. 

class Shape {
  float xPosition, yPosition, xSpeed, ySpeed;
  color c;
  float size;

  Shape() {
    xPosition = width / 2;
    yPosition = height / 2;
    c = color(random(255), random(255), random(255));
    size = random(10, 30);
    xSpeed = random(5);
    ySpeed = random(5);
  }

  Shape(float xPosition, float yPosition, color c, float size) {
    this.xPosition = xPosition;
    this.yPosition = yPosition;
    this.c = c;
    this.size = size;
    xSpeed = random(5);
    ySpeed = random(5);
  }

  void move() {
    xPosition += xSpeed;
    yPosition += ySpeed;
  }
}

class Ellipse extends Shape {
  Ellipse(float xPosition, float yPosition, color c, float size) {
    super(xPosition, yPosition, c, size);
    ySpeed = 0;
  }

  void drawShape() {
    fill(c);
    ellipse(xPosition, yPosition, size, size);
  }

  void move() {
    xPosition += xSpeed;
    if (xPosition > width) {
      xPosition = -width / 4;
    }
  }
}

Your job is to add another child class of Shape. For example:

  • Rectangle
  • Triangle
  • Square

Your new subclass should:

  • call super(...) in its constructor,
  • have its own drawing method,
  • override move() in some visible way.

Go-Getter Option

If you finish early, try creating an array that stores different child objects in one array. For example, you might decide that even indexes will store Ellipse objects and odd indexes will store your new shape objects.

This is meant to be the challenging part, so you should design the details yourself. Think about these questions:

  • What type should the array be?
  • Where should you create the objects?
  • How can you alternate between two different subclasses?
  • How will you update and draw all the shapes in one loop?

Small hint: if both child classes inherit from Shape, then one array of Shape references can store both kinds of objects.

  • When you call shapes[i].move(), which version of move() is executed for each object?
  • Explain how the program decides which method to use.

This is a first step toward polymorphism: one array can store objects of different child classes because all of them are also Shape objects.

What to Submit

  • Your full sketch folder.
  • Clear comments showing what is inherited and what is overridden.
  • A short note at the top saying whether you chose Option A or Option B.

Checklist

  • I used extends.
  • I used super(...) in the child constructor.
  • I overrode at least one method.
  • My child class looks or moves differently from the parent idea.
  • My code is commented and organized.

Reminder

Inheritance is not just copying code into a second class. The point is to keep the common parts in one parent class, then let child classes specialize the behavior.
