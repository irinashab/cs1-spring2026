Objects and Classes

Think about people. We all have the same kinds of properties: name, age, height. But each person has different values.

Key idea:
A class is a blueprint.
An object is an instance created from that blueprint.

In programming, a class groups together:

  • Variables (data / state)
  • Functions (behavior / methods)

Minimal Class Structure

class Ball {
  // variables (state)

  // constructor

  // methods (behavior)
}

Creating an Object

We use the keyword new:

Ball myBall;

void setup(){
  myBall = new Ball();
}
The constructor runs automatically when you use new.

Using an Object (Dot Syntax)

myBall.move();
myBall.display();
Dot syntax:
object.method() means:
“ask this specific object to perform an action”

Constructors

A constructor is a special function that runs automatically when you create an object using new. 

Ball b = new Ball();

When this line runs, the constructor is executed.

How to recognize a constructor

class Ball {

  Ball() {   // constructor
    // initialization code
  }

}

A constructor has two key properties:

  • It has the same name as the class
  • It has no return type — not even void
A constructor is how an object is born.

Constructor vs. Method

class Ball {

  Ball() {        // constructor
    xpos = 100;
  }

  void move() {   // regular method
    xpos += 2;
  }

}
Feature Constructor Method
Name matches class Yes No
Has return type No Yes
Called with new Yes No
Called with dot syntax No Yes

Common mistakes

void Ball() { }          // not a constructor
BallConstructor() { }   // not a constructor
b.Ball();               // not how constructors are called

Full Ball Class

class Ball {
  float xpos, ypos;
  float xspeed, yspeed;
  color ballColor;
  int diameter;

  Ball(){
    xpos = random(width);
    ypos = random(height);
    xspeed = 2;
    yspeed = 3;
    ballColor = color(0);
    diameter = 20;
  }

  void move(){
    xpos += xspeed;
    ypos += yspeed;
  }

  void display(){
    fill(ballColor);
    ellipse(xpos, ypos, diameter, diameter);
  }
}

Custom Constructor

Ball(float xpos, float ypos, color ballColor,
     int diameter, float xspeed, float yspeed){

  this.xpos = xpos;
  this.ypos = ypos;
  this.ballColor = ballColor;
  this.diameter = diameter;
  this.xspeed = xspeed;
  this.yspeed = yspeed;
}
Important:
this.xpos refers to the object's variable.
xpos is the parameter passed in.

Now we can create different balls:

Ball b1 = new Ball(100, 200, color(255, 0, 0), 40, 3, 2);
Ball b2 = new Ball(300, 50, color(0, 0, 255), 60, -2, 3);

Multiple Objects

Ball b1, b2;

void setup(){
  size(600,400);
  b1 = new Ball(100, 200, color(255, 0, 0), 40, 3, 2);
  b2 = new Ball(400, 150, color(50, 100, 220), 60, -2, 3);
}

void draw(){
  background(255);
  b1.move();
  b1.display();
  b2.move();
  b2.display();
}
Each object has its own data — changing one object does not change the other.

Before vs After

  • Before: many variables like x1, y1, x2, y2
  • After: one class → many objects

Naming Conventions

  • Classes: Ball, GameCharacter
  • Variables/methods: camelCase
  • Constants: ALL_CAPS
