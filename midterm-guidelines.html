Midterm Project

Midterm Project Guidelines

You may build on your previous work or start a completely new project. Your program must include variables, functions, and arrays and/or loops. You are encouraged to enhance your program with images, fonts, sound, or video if they improve the overall experience.

Grading Criteria

1. Code

  • Use clear and meaningful names for variables and functions.
  • Write code that is organized, efficient, and well commented.
  • Use functions and loops instead of repeating code.
  • Use arrays when they are helpful.
  • Your program must compile and run correctly.
  • Avoid hard-coding numbers when possible; use variables and screen dimensions.

2. Idea / Short Write-Up

Submit a short write-up (1–2 paragraphs) describing your project.

  • Explain what your program does.
  • Describe how you approached building it.
  • Mention any challenges you encountered.
  • Explain how you solved those challenges.

Your project must be original work.

3. Functionality & Design

  • Your program should be easy to understand and easy to use.
  • Follow the K.I.S.S. principle (Keep It Simple, Stupid).
  • The interface should be clear and easy to navigate.
  • Thoughtful use of colors, fonts, images, or sound can improve the user experience.
