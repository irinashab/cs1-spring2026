Lesson 3: Variables, State, and Interaction

Big idea

Up to now, we have been drawing shapes by typing in numbers. Today we will learn how to:

  • name important numbers using variables,
  • change values over time,
  • understand why motion and interaction work in Processing.

By the end of this lesson, your sketches should feel less fragile and more intentional.

1. Numbers vs. ideas

Consider this sketch: 

ellipse(250, 150, 100, 100);

This works, but what do these numbers mean?

  • Is 250 the center?
  • Is 100 the radius or the diameter?
  • What happens if the canvas size changes?

The computer does not know what these numbers represent. We do — but only in our heads.

2. Variables: naming what matters

Now look at the same idea written with variables: 

int centerX = width / 2;
int centerY = height / 2;
int diameter = 100;

ellipse(centerX, centerY, diameter, diameter);

Visually, nothing changed. What changed is that we wrote down our intentions.

A variable is just a name for a value that the computer should remember.

Variables:

  • make code easier to read,
  • make drawings easier to change,
  • reduce mistakes when projects get larger.

This is not about typing fewer numbers. It is about making ideas visible.

3. A quick note on data types

When we create a variable, we must also say what kind of value it stores. This is called the variable’s data type.

Common data types in Processing

  • int – whole numbers (e.g. 0, 42, -7)
  • float – numbers with decimals (e.g. 3.5, 0.25)
  • String – text (e.g. "hello")

Examples: 

int count = 10;
float speed = 2.5;
String message = "Hello, Processing";

For now:

  • use int for positions, sizes, and counters,
  • use float when you need smoother motion or fractional values,
  • use String for text and messages.

We will see more data types later, but these three will take us a long way.

4. Variables can change over time

So far, our variables have fixed values. But variables can also change while the program runs.

This is the key idea behind motion. 

int y = 200;

void draw() {
  background(255);
  ellipse(250, y, 60, 60);
  y = y - 1;
}

Read this in English:

  • y starts at 200,
  • each frame we draw the circle at the current y,
  • then we update y for the next frame.

The line 

y = y - 1;

means:

Take the old value of y, subtract 1, and store it back in y.

This is how motion happens.

5. State: remembering values

The pattern 

x = x + 1;

introduces a new idea called state.

The value of x now depends on what it was before. The program remembers information from one frame to the next.

This is why Zoog can “go to heaven”:

  • Zoog’s position is stored in a variable,
  • that variable changes a little each frame,
  • the drawing updates automatically.

We are no longer drawing a single picture. We are describing how a picture changes over time.

6. Built-in variables: the mouse is not magic

Processing already keeps track of some changing values for us. For example: 

mouseX
mouseY

These are just variables whose values are updated automatically. 

ellipse(mouseX, mouseY, 40, 40);

There is no new concept here. We are simply using variables that Processing updates for us.

You can even rewrite it as: 

int x = mouseX;
int y = mouseY;
ellipse(x, y, 40, 40);

Now it looks ordinary — and that is a good thing.

7. Previous values and motion speed

Processing also remembers the mouse position from the previous frame: 

pmouseX
pmouseY

This allows us to measure change: 

float speed = abs(mouseX - pmouseX);
strokeWeight(speed);
line(pmouseX, pmouseY, mouseX, mouseY);

In English:

  • the faster the mouse moves, the bigger the difference,
  • the bigger the difference, the thicker the line.

This idea is used in the changingLines sketch.

8. Randomness and debugging

Sometimes we want variation instead of precision. Processing provides: 

random(min, max)

which returns a random float. 

int r = (int)random(256);
int g = (int)random(256);
int b = (int)random(256);
background(r, g, b);

When things behave unexpectedly, we can always check values with: 

println(r, g, b);

Printing values is one of the simplest and most useful debugging tools.

9. A brief note on events (optional)

Processing can also respond to discrete events, such as a mouse click: 

void mousePressed() {
}

We will use events soon. For now, think of them as another way values can change.

Takeaway

  • Variables let us name ideas.
  • Some variables change automatically.
  • Some variables change because we update them.
  • Motion is just small changes over time.

Once variables make sense, interaction and animation stop feeling magical.

Looking ahead
