Simple Animations in Processing

Animation is created by displaying a sequence of images quickly, one after another. When the images change rapidly, the eye perceives motion.

This idea goes back to the early work of photographers such as Eadweard Muybridge (1830–1904), who studied motion by taking many photographs in rapid succession.

In Processing, we can create animation by:

  • loading a sequence of images
  • storing them in an array
  • displaying one image per frame
  • cycling through the images repeatedly

Example: Muybridge Animation

The following sketch displays a sequence of images that repeat in a loop, creating the illusion of motion. 


/* 
This sketch is a tribute to Eadweard Muybridge (1830–1904),
the English photographer known for his studies of motion.

The images are taken from his work "Animal Locomotion".

IS
Sept 28, 2025
*/

int numFrames = 12;
int frame;

PImage [] imgs = new PImage[numFrames];

void setup(){

  size(510,325);

  frameRate(8);

  // load all images
  for(int i = 0; ii<numFrames; i++){
    imgs[i] = loadImage("frame" + i + ".png");
  }

}

void draw(){

  frame = frameCount % numFrames;

  image(imgs[frame],0,0,width,height);

}

Step 1: Store images in an array


int numFrames = 12;
PImage [] imgs = new PImage[numFrames];

Each element of the array stores one frame of the animation.

Step 2: Load the images

Inside setup() we load the images into the array. 


for(int i = 0; i i< numFrames; i++){
  imgs[i] = loadImage("frame" + i + ".png");
}

This automatically loads:

  • frame0.png
  • frame1.png
  • frame2.png
  • ...
  • frame11.png

All image files must be placed inside the sketch’s data folder.

Step 3: Control animation speed


frameRate(8);

This sets the animation speed to about 8 frames per second.

Step 4: Choose which image to display


frame = frameCount % numFrames;

frameCount increases every frame.

The modulo operator % keeps the frame number between 0 and numFrames-1, so the animation loops.

Step 5: Display the image


image(imgs[frame],0,0,width,height);

This draws the current animation frame.

Animation Pattern

Many simple animations follow this pattern: 


load frames
↓
choose frame index
↓
display frame
↓
repeat

Common Mistakes

  • Images are not in the data folder
  • File names do not match the code
  • The number of frames does not match numFrames
  • The animation speed is too fast or too slow

Mini Exercise

  1. Change the animation speed using frameRate().
  2. Try using only 6 frames instead of 12.
  3. Modify the code so the animation plays forward and then backward.

Keyboard-Controlled Sprite Animation

In games, animation often depends on user input. For example, a character may animate only when it moves.

In the following sketch, pressing the right arrow key moves the character and advances the animation. 


int numFrames = 6;
PImage[] walk = new PImage[numFrames];

int frame = 0;
int x = 100;

void setup(){

  size(600,300);

for(int i=0;i<numFrames;i++){
    walk[i] = loadImage("walk"+i+".png");
}

}

void draw(){

  background(240);

  image(walk[frame],x,150);

}

void keyPressed(){

  if(keyCode == RIGHT){

    x += 10;

    frame = (frame + 1) % numFrames;

  }

}

How it works

  • The array walk[] stores the animation frames.
  • Each time the right arrow key is pressed, the character moves.
  • The frame index increases by 1.
  • The modulo operator keeps the animation looping.

Why this pattern is useful

This technique is used in many games:

  • walking animations
  • running animations
  • character attacks
  • jumping sequences

Instead of drawing one static image, the program cycles through a sequence of frames to simulate motion.

Summary

  • Animation is created by displaying a sequence of images.
  • Images are usually stored in an array.
  • The current frame is selected using a frame index.
  • The modulo operator allows the animation to loop.
  • Input (keyboard or mouse) can control when animation advances.
