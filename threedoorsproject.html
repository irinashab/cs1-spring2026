Overview

This is a group project in which you will design an interactive story game using Processing.

The player begins in front of three doors. Clicking on different doors (or areas of the screen) takes the player to different “rooms,” each with its own story and outcome.

Your program should use conditional statements to control the narrative flow, determining which scenes and events occur based on the player’s choices.

Learning Goals

By completing this project, you will practice:

  • Using if, else if, and else to control program flow
  • Combining conditions using && and || when appropriate
  • Tracking state (what room or scene the player is currently in)
  • Designing an interactive program with multiple possible outcomes

Requirements

Your project must meet the following requirements:

1) Group work

  • Collaborate on story design, visuals, and code.
  • Divide responsibilities clearly.

2) Three doors

  • Display three clickable doors or regions on the screen.
  • Clicking a door must move the player to a different room or scene.

3) State management

  • Use a variable (for example, int room, int scene, or similar) to keep track of where the player is.
  • At any moment, the player should be in exactly one room.

4) Return to the starting screen

  • The player must be able to return to the starting screen with the three doors.
  • This may happen through a button, a mouse click, or a key press.
  • Returning to the start should reset the story state appropriately.

5) Conditionals

  • Use if, else if, and else to control which scenes and events occur.
  • You may use nested if statements.
  • You may also use && and || to combine conditions.

6) Multiple story paths

  • Your story must have at least three distinct paths.
  • Each path should lead to one or more possible endings.

7) Sound

  • Incorporate at least two sound effects (for example: door creaks, footsteps, ambient sounds).
  • Sound effects may be simple and short.

8) Images (optional)

  • You may insert images instead of creating your own drawings.

Media (Images & Sound)

Sound Library

  1. Open your sketch.
  2. Select Sketch → Import Library → Add Library.
  3. Search for Sound.
  4. Install the Sound library from the Processing Foundation.

See the soundDemo sketch for playback details. You may use placeholder sounds while developing your logic.

How to Add Media Files to Your Project

To add a media file (an image, a sound file, etc.) to your project:

  1. Open your sketch.
  2. Select Sketch → Add File... → navigate to your file → click Open.
  3. Or simply drag the file onto the open sketch window.

This will create a data folder in your project. All media files must live inside data.

How to Display an Image on the Canvas

Processing supports these image types: .gif, .jpg, .tga, .png.

See the imageDemo sketch for details.

Images (Optional, but Read Carefully)

If you use images or sound, your sketch depends on external files. When submitting, make sure you submit the entire sketch folder (including the data folder), not just the .pde file.

Design Notes (Read Carefully)

  • You are not expected to use advanced logic or complex math.
  • Focus on clear decisions and readable code.
  • If your logic becomes deeply nested and hard to read, that is a sign to:
    • reorganize your conditions, or
    • combine them using && or ||.

      • Think of the starting screen as just another “room” in your program.

Submission & Presentation

  • Submit one Processing sketch per group.
  • Make sure all group members’ names appear at the top of the file.
  • Groups will present their projects in class on Friday, February 13 (this deadline is negotiable).

During the presentation, be prepared to briefly explain:

  • how conditionals control the story flow

Final Reminder

There is no single “correct” solution. What matters is that your program makes decisions clearly and consistently.

Quick Reference (Demos)

You may copy and adapt these patterns in your lab and project.

1) Images (must be in data)

PImage door;

void setup() {
  size(600, 400);
  door = loadImage("door.png");  // file must be in data
}

void draw() {
  image(door, 0, 0);
}

2) Choosing what to draw (state + conditionals)

int room = 0;

void draw() {
  if (room == 0) {
    background(200);
    text("Start screen", 50, 50);
  } else if (room == 1) {
    background(100);
    text("Room 1", 50, 50);
  }
}

3) Mouse inside rectangle (&&)

if (mouseX > x &&
    mouseX < x + w &&
    mouseY > y &&
    mouseY < y + h) {
  println("Mouse is inside the rectangle");
}

4) Multiple triggers (||)

if (key == 'r' || key == 'R') {
  room = 0; // return to start
}

5) Sound (optional)

import processing.sound.*;

SoundFile ding;

void setup() {
  size(400, 300);
  ding = new SoundFile(this, "ding.mp3");
}

void mousePressed() {
  try {
    ding.play();
  } catch (Exception e) {
    println("Sound could not be played.");
  }
}

One design rule

At any moment, the program should be in exactly one state (one room, one screen, one situation).
