Lesson 10: Introduction to Arrays in Processing

What is an Array?

An array is a structured collection of elements of the same data type, stored in contiguous memory locations and accessed through a single reference (variable).
For example, suppose you want to store the scores of five students. Instead of writing:

int score1 = 85;
int score2 = 90;
int score3 = 78;
int score4 = 92;
int score5 = 88;

you can write more efficiently

int[] scores = {85, 90, 78, 92, 88};

What Is an Array Conceptually?

An array is one variable that stores many values, all of the same type.

Instead of writing:


int a = 3;
int b = 7;
int c = 2;
int d = 9;
int e = 5;

We can write:


int[] numbers = {3, 7, 2, 9, 5};

Now we have one variable (numbers) that contains five values.

Think of an array as a row of labeled boxes in memory:


Index:   0    1    2    3    4
Value:   3    7    2    9    5

The number above each value is its index.

Arrays start counting from 0.

Declaring and Initializing an Array

There are multiple ways to declare and initialize an array in Processing.

1. Declaring an Empty Array

Before storing values, you need to declare an array:

int[] numbers;  // Declares an integer array

At this point, numbers does not yet have a size.

2. Creating an Array with a Fixed Size

To allocate memory for the array, use the new keyword:

numbers = new int[5];  // Creates an array with 5 slots (all initially 0)

All elements in this array are initially set to 0 (for numeric types).

3. Initializing an Array with Specific Values

int[] numbers = {3, 7, 2, 9, 5};  // An array with predefined values

Processing will automatically determine the size of the array (in this case, 5).

Accessing Array Elements

Each element in an array is stored at an index (position), starting from 0.

int[] numbers = {3, 7, 2, 9, 5};

println(numbers[0]);  // Output: 3 (first element)
println(numbers[2]);  // Output: 2 (third element)

If you try to access an index out of range, Processing will give an ArrayIndexOutOfBoundsException error.

Example of an error:

println(numbers[5]);  // Error: Index 5 is out of bounds (valid range is 0–4)

Using arr.length in Processing

In Processing each array ‘knows’ how long it is, the .length property is used to determine the number of elements in an array. This is essential for loops, conditionals, and general array operations. The syntax is

arrayName.length

Here

  • arrayName is the name of the array.
  • .length returns the number of elements in the array (not the highest index!).

Important Notes:

  1. length is a property, not a method, so no parentheses are needed (arr.length, not arr.length()).
  2. It gives the total number of elements, so the highest valid index is arr.length - 1.

    3. Common Beginner Mistake: The Off-By-One Error

    One of the most common mistakes when working with arrays is using <= instead of < in a loop.

    
int[] numbers = {10, 20, 30, 40, 50};

for (int i = 0; i <= numbers.length; i++) {
    println(numbers[i]);   // What happens?
}

    This causes an ArrayIndexOutOfBoundsException.

    Why?

    • numbers.length is 5.
    • Valid indices are 0, 1, 2, 3, 4.
    • When i becomes 5, the program tries to access numbers[5], which does not exist.

    Correct version:

    
for (int i = 0; i < numbers.length; i++) {
    println(numbers[i]);
}

    Remember:
    Use < not <= when looping through arrays.

Basic Example
Let’s create an array and use length to print its size.

int[] numbers = {10, 20, 30, 40, 50};
println("Array size: " + numbers.length);  // Output: 5

Using .length in Loops
A common use of .length is looping through all elements of an array.

int[] numbers = {3, 7, 2, 9, 5};
for (int i = 0; i < numbers.length; i++) {  
    println("Element at index " + i + ": " + numbers[i]);  
    //Since length = 5, the loop runs from i = 0 to i = 4, avoiding out-of-bounds errors.
}

Some Examples

Example 1: Drawing Multiple Circles with an Array

int[] xPositions = {50, 100, 150, 200, 250};
//This program draws five circles at different x-positions stored in an array.
void setup() {
  size(300, 200);
}

void draw() {
  background(220);
  
  for (int i = 0; i < xPositions.length; i++) {
    ellipse(xPositions[i], height / 2, 30, 30);
  }
}

Example 2: Random Color Array

//This sketch creates an array of colors and randomly assigns colors to circles.
color[] colors = {color(255, 0, 0), color(0, 255, 0), color(0, 0, 255), color(255, 255, 0), color(255, 0, 255)};

void setup() {
  size(400, 200);
  noLoop();  // Ensures it only runs once
}

void draw() {
  background(220);
  
  for (int i = 0; i < colors.length; i++) {
    fill(colors[int(random(colors.length))]);  // Choose a random color from the array
    ellipse(i * 80 + 50, height / 2, 50, 50);
  }
}

Example 3: Moving Objects with Arrays

//This example creates multiple moving circles using an array.
int[] xPositions;
int numCircles = 10;

void setup() {
  size(500, 200);
  xPositions = new int[numCircles];  // Initialize the array
  
  for (int i = 0; i < numCircles; i++) {
    xPositions[i] = int(random(width));  // Assign random x-positions
  }
}

void draw() {
  background(220);
  
  for (int i = 0; i < numCircles; i++) {
    ellipse(xPositions[i], height / 2, 30, 30);
    xPositions[i] += 2;  // Move circles to the right
    
    // Reset when they reach the edge
    if (xPositions[i] > width) {
      xPositions[i] = 0;
    }
  }
}

For Go-Getters: Why Do Arrays Start at Index 0?

In most programming languages, including Processing (Java-based), arrays start indexing from 0 instead of 1. This might seem unusual at first, but there’s a good reason for it—actually, several reasons! Let’s break it down:

1. Memory and Address Calculation

Computers store arrays as a continuous block of memory. Each element in an array is stored sequentially in memory. To access an element at a specific index, the computer needs to calculate its memory address.

Assume we have an integer array:

int[] numbers = {10, 20, 30, 40, 50};

Each integer typically takes 4 bytes in memory (on most systems). If the array starts at memory location 1000, the addresses of the elements will be:

Index Value Memory Address
0 10 1000
1 20 1004
2 30 1008
3 40 1012
4 50 1016

The memory address of an element at index i is calculated as:

Base Address+(Index×Size of each element)

For example, to get numbers[2], the address calculation is:

1000 + (2 × 4) = 1008

By starting at index 0, the formula remains:

Base Address + (Index × Size)

If we started at 1, we’d need to adjust the formula:

Base Address + ((Index - 1) × Size)

This would add unnecessary complexity to every array access operation.

2. Consistency with Pointer Arithmetic

Many early programming languages, including C (which influenced Java and Processing), used pointers to reference memory locations. In C, an array is essentially a pointer to the first element.

 int numbers[] = {10, 20, 30, 40, 50};

The name numbers itself represents the memory address of the first element (numbers[0]). Using pointer arithmetic, we can access other elements:

*(numbers + 1)  // Access numbers[1]
*(numbers + 2)  // Access numbers[2]

Since memory addressing naturally works with offsets from the first element, starting at 0 makes the most sense.

3. Simplicity in Looping and Iteration

Starting from 0 makes loops simpler and more intuitive.
For example, looping through an array:

for (int i = 0; i < numbers.length; i++) {
    println(numbers[i]);
}

If arrays started at 1, the loop would need to be:

    println(numbers[i - 1]);  // Adjust for 1-based indexing
}

The i - 1 correction adds an unnecessary extra step.

4. Historical Influence

Many early programming languages, like C and Assembly, used zero-based indexing due to memory and performance reasons. Since Java (and Processing) evolved from C-like languages, it inherited this behavior.

However, some languages do use 1-based indexing:

  • MATLAB
  • Lua
  • Fortran
  • R

These languages were designed more for scientific computation and human-friendly readability, rather than low-level memory efficiency.

Final Thought

While humans naturally start counting from 1, computers think in terms of memory offsets, which makes starting at 0 the best choice! Sometimes I wish humans would also start counting at zero!

Looking Ahead: 2D Arrays

If a 1D array is like a row of boxes, a 2D array is like a grid (rows and columns).


int[][] grid = new int[3][4];

This creates a table with 3 rows and 4 columns.

We will use 2D arrays later for things like:

  • Grids
  • Games
  • Pixel-based graphics
  • Board simulations

For now, focus on mastering 1D arrays first.
