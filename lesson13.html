Lesson: Images and Pixels

Big idea

Shapes, images, and animations are all made of pixels. A line does not magically appear because we call line(). At the lowest level, the computer colors many individual pixels so that our eyes see a line. Today we will work at that lower level.

Goals

  • Understand that the screen is a grid of pixels.
  • Use Processing's built-in pixels[] array.
  • Convert between 2D coordinates and a 1D array index.
  • Understand the color datatype and helper functions such as red(), green(), blue(), and brightness().
  • Read pixel data from a PImage and write new pixel colors to the screen.
  • See why get() and set() are convenient, but too slow for serious image processing.

1. Pixels on the screen

Processing gives us access to the window's pixels through a built-in array called pixels[]. Each element stores the color of one pixel.

Important: before reading or writing the screen pixel array, call loadPixels(). After you finish changing it, call updatePixels(). 
size(200, 200);
loadPixels();

for (int i = 0; i < pixels.length; i++) {
  pixels[i] = color(random(255));
}

updatePixels();

This fills the entire canvas with random grayscale values.

2. A 2D screen stored in a 1D array

The screen is two-dimensional, but pixels[] is one-dimensional. To find the correct location, use this formula:

Index formula: loc = x + y * width

Here, x is the column, y is the row, and width is the width of the window. 

loadPixels();

for (int x = 0; x < width; x++) {
  for (int y = 0; y < height; y++) {
    int loc = x + y * width;

    if (x % 2 == 0) {
      pixels[loc] = color(255);
    } else {
      pixels[loc] = color(0);
    }
  }
}

updatePixels();

This creates vertical black-and-white stripes.

3. The color datatype

In Processing, a color is stored as a single value of type color. Even though it is stored internally as an integer, we usually treat it as a special datatype for color information. 

color c1 = color(255, 0, 0);    // red
color c2 = color(0, 255, 0);    // green
color c3 = color(0, 0, 255);    // blue
color c4 = color(128);          // gray
color c5 = color(255, 0, 0, 120); // transparent red

The helper functions red(), green(), blue(), and brightness() let us pull information back out of a color. 

color c = img.pixels[loc];
float r = red(c);
float g = green(c);
float b = blue(c);
float br = brightness(c);
Note: color() accepts floats too. If you write color(gray) and gray is a float, Processing converts it to a color value for you.

4. Helper functions get() and set()

Processing provides helper functions for reading and writing individual pixels: 

color c = get(mouseX, mouseY);
set(10, 20, color(255, 0, 0));

These functions are easy to understand and are good for small demonstrations.

But: get() and set() are much slower than working directly with pixels[]. For image processing inside nested loops, use pixels[] instead.

Example idea: sample a color under the mouse and paint a large rectangle with that color.

5. Working with a PImage

A PImage object stores its own width, height, and pixel array. 

PImage img;

void setup() {
  size(300, 168);
  img = loadImage("hedgehog.jpeg");
}

If we want to read pixels from the image, we must also call img.loadPixels(). 

void draw() {
  loadPixels();
  img.loadPixels();

  for (int x = 0; x < width; x++) {
    for (int y = 0; y < height; y++) {
      int loc = x + y * width;

      float r = red(img.pixels[loc]);
      float g = green(img.pixels[loc]);
      float b = blue(img.pixels[loc]);

      pixels[loc] = color(r, g, b);
    }
  }

  updatePixels();
}

This displays the image without calling image(). That may seem unnecessary, but it gives us a place to insert our own pixel-processing code.

6. Canonical pixel-processing pattern

Most of our image-processing examples will follow this same structure:

loadPixels();
img.loadPixels();

for (int x = 0; x < img.width; x++) {
  for (int y = 0; y < img.height; y++) {
    int loc = x + y * img.width;

    // 1. Read a color from the image
    color c = img.pixels[loc];

    // 2. Extract the components
    float r = red(c);
    float g = green(c);
    float b = blue(c);

    // 3. Process them
    // ... change r, g, b here ...

    // 4. Write the new color to the screen
    pixels[loc] = color(r, g, b);
  }
}

updatePixels();

7. Example: grayscale

To convert a color pixel to grayscale, we can average its red, green, and blue components. 

float gray = (r + g + b) / 3.0;
pixels[loc] = color(gray);

Since grayscale uses one value, Processing copies that value into the red, green, and blue channels.

8. Example: adjusting brightness

We can make each pixel brighter or darker by multiplying its RGB values by a factor. 

float adjustBrightness = map(mouseX, 0, width, 0, 2);

r *= adjustBrightness;
g *= adjustBrightness;
b *= adjustBrightness;

r = constrain(r, 0, 255);
g = constrain(g, 0, 255);
b = constrain(b, 0, 255);

pixels[loc] = color(r, g, b);

A more interesting variation is to use distance from the mouse, creating a flashlight effect.

9. Example: keep one color family

We can compare each pixel to a target color. Pixels close to that target stay colorful, while all others are turned to grayscale. 

color c = img.pixels[i];
float r = red(c);
float g = green(c);
float b = blue(c);

float d = dist(r, g, b, red(target), green(target), blue(target));

if (d > threshold) {
  float gray = (r + g + b) / 3.0;
  img.pixels[i] = color(gray);
}

This is a nice example of how image processing can be used artistically, not just technically.

10. Example: pointillism

Instead of changing image pixels directly, we can use image colors to draw shapes. In the pointillism example, we choose a random pixel, read its color, and use that color to draw a circle. 

int x = int(random(img.width));
int y = int(random(img.height));
int loc = x + y * img.width;

float r = red(img.pixels[loc]);
float g = green(img.pixels[loc]);
float b = blue(img.pixels[loc]);

fill(r, g, b, 100);
ellipse(x, y, pointillize, pointillize);

This brings us back to shapes, but now the image acts like a color database.

Discussion questions

  1. Why does the formula x + y * width make sense?
  2. Why do we need loadPixels() and updatePixels()?
  3. Why are get() and set() convenient but not ideal for heavy image work?
  4. Why does color(gray) produce a grayscale color?
  5. What changes if the image size and the canvas size do not match?
  6. When would you write pixels directly, and when would you instead use pixel colors to draw shapes?

In-class exercises

  1. Fill the canvas with random grayscale values.
  2. Create vertical stripes by setting even columns to white and odd columns to black.
  3. Display an image without calling image().
  4. Convert an image to grayscale.
  5. Create a brightness slider using the mouse.
  6. Create a flashlight effect based on distance from the mouse.
  7. Choose a target color and keep only that color family.
  8. Use image colors to create a pointillist drawing.

Go-getter challenge

Can you create a filter that reacts differently in different parts of the image? For example, brighten only pixels near the mouse, darken pixels far away, or keep warm colors while muting cool colors.

Practice Tasks

  1. Channel swap: Modify your pixel-processing code so that the red and green channels are switched. In other words, if a pixel originally has values (r, g, b), the new pixel should become (g, r, b).
  2. Mirror the image: Create a horizontally mirrored version of the image. Hint: when you are at pixel (x, y), think about which pixel should be read from the opposite side of the row.
  3. Reflection: Write a few sentences answering the question: What surprised you most about working directly with pixels? You may also mention what felt confusing at first, what became clearer, or which visual effect you found most interesting.
