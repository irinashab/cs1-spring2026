Midterm Review Sheet

CS1 Midterm Review Sheet

Use this as a checklist. If you hesitate on any section, review that topic.

1) Coordinates & Drawing

  • width and height
  • Center: (width/2, height/2)
  • Top-left corner: (0,0)
  • rect(x,y,w,h) uses top-left corner
  • ellipse(x,y,w,h) uses center

Common mistake: confusing the anchor point for rectangles vs ellipses.

2) Conditionals

  • == compares; = assigns
  • >=, <=, !=, &&, ||
  • Double negation: !(!x) equals x

Common mistake: using = inside a condition or not evaluating the condition first.

3) Variables & Types

  • Types: int, float, boolean, String
  • camelCase naming
  • Updates: x--, x -= 4, x += 3
  • Be able to draw a simple memory diagram

Common mistake: forgetting that assignment overwrites the old value.

4) Scope

  • Global variables: declared outside functions, visible everywhere
  • Local variables: declared inside a block { }, visible only there
  • A variable declared in setup() is not visible in draw()

Be able to answer: Will it compile? If yes, what prints? If not, why?

5) Loops

For loop template:

for (initialization; condition; update) {
  // body
}

While loop template:

while (condition) {
  // body
}
  • Write loops for even/odd ranges
  • Watch loop bounds (off-by-one)
  • Be alert for the accidental semicolon: for(...);

6) Functions

  • Parameter vs argument
  • Return value and return
  • Local variables inside a function
  • Describe a function in one sentence (“big picture”)

7) Arrays

  • Indexing starts at 0
  • Last valid index is arr.length - 1
  • arr.length is the size
  • Nested indexing: arr[arr[4]]

Random element:

arr[int(random(arr.length))]

Common mistake: going out of bounds.

8) Random Numbers

  • random(n) returns a float in [0, n)
  • Die roll (1..12): int(random(12)) + 1

Common mistake: forgetting + 1.

9) Compile-Time vs Run-Time Errors

  • Compile-time: scope errors, missing semicolons, undefined variables
  • Run-time: array index out of bounds, division by zero

How to know you’re ready

  • You can trace code without guessing.
  • You can draw memory diagrams clearly.
  • You can spot a for(...); bug instantly.
  • You can explain why code does or does not compile.
  • You can handle nested array indexing confidently.
