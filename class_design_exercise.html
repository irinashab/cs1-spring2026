Class Design Exercise

In this exercise, you will design your own class, create several objects from it, and animate them on the screen.

Reminder: A class is the blueprint, and an object is an instance created from that blueprint.

Step 1: Class Design & Planning

Choose an object that has clear properties and behaviors.

Ask yourself:

  • What does my object represent?
  • What properties (instance variables) does it have?
  • What behaviors (methods) should it perform?
  • How will it look on the screen?
Pick something that is visually interesting but still manageable: a butterfly, fish, ghost, robot, spaceship, snowflake, bouncing creature, etc.

Example Design

CLASS NAME: FlutteringButterfly

DESCRIPTION: A butterfly that flutters around the screen

Properties

  • float x, y — position
  • float wingAngle — current wing position
  • color bodyColor, wingColor
  • float speedX, speedY — movement direction
  • float size — scale of butterfly

Methods

  • void flutter() — makes the wings flap
  • void move() — updates position
  • void display() — draws the butterfly
  • void checkBounds() — keeps it on the screen

Step 2: Implementation

File Structure

  • Main tab: contains setup(), draw(), and your object declarations
  • Class tab: contains your custom class
In Processing, put your class in a separate tab and give that tab the same name as the class.

Implementation Checklist

  • Create your class using proper naming convention (PascalCase)
  • Declare all instance variables
  • Write at least one constructor
  • Implement your planned methods
  • Test that one object works correctly before making more

Step 3: Create & Animate Multiple Objects

Create at least 3 instances of your class.

Make them different by changing their:

  • starting positions
  • colors
  • sizes
  • speeds or directions
The objects should come from the same class, but have different initial values.

Step 4: Go-Getter Extension

Create an array of objects instead of declaring each object separately.

Ball[] balls = new Ball[10];

Then use a loop to create and display all of them.

Minimum Requirements

  • one custom class
  • at least 3 objects
  • at least 3 instance variables
  • at least 3 methods
  • at least one constructor
  • visible animation

Suggestions for Success

  • Start simple. Get one object working first.
  • Make display() work before worrying about fancy movement.
  • Use meaningful variable names.
  • Do not make your design too complicated too soon.
  • Save often and test often.
